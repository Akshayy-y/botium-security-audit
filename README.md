Project Documentation
Internal Security Audit – Botium Toys
🔹 1. Overview

This project is part of the Google Cybersecurity Professional Certificate.
The objective of this activity was to perform an internal security audit for a fictional company, Botium Toys, to identify risks, vulnerabilities, and compliance gaps.

🔹 2. Objective
Evaluate the organization’s current security posture
Identify missing security controls
Assess compliance with standards like PCI DSS and GDPR
Provide recommendations to improve security
🔹 3. Scope of Audit

The audit covered:

Employee devices (laptops, smartphones)
Internal network and internet access
Business systems (database, ecommerce, inventory)
Data storage and retention
Legacy systems
🔹 4. Methodology

The audit was conducted using concepts from the National Institute of Standards and Technology Cybersecurity Framework (NIST CSF).

Steps followed:

Identified assets
Reviewed existing controls
Analyzed risks and vulnerabilities
Completed controls and compliance checklist
Provided recommendations
🔹 5. Key Findings
Security Issues Identified
No implementation of least privilege access control
Lack of data encryption for sensitive information
No intrusion detection system (IDS)
Absence of backup and disaster recovery plans
Weak password policies and no password management system
Existing Controls
Firewall configured
Antivirus software installed
Physical security (locks, CCTV, fire systems)
Data integrity and availability maintained
🔹 6. Risk Assessment
Risk Score: 8/10 (High Risk)
High exposure to:
Data breaches
Financial loss
Regulatory penalties
🔹 7. Compliance Analysis
❌ PCI DSS (Payment Security)
No encryption
No restricted access to card data
Weak password controls
❌ GDPR (EU Data Protection)
Data protection measures insufficient
No proper data classification
SOC Compliance
Data integrity and availability maintained
Weak confidentiality and access control
🔹 8. Recommendations
Implement least privilege & role-based access control
Enable encryption for sensitive data
Deploy Intrusion Detection System (IDS)
Establish backup and disaster recovery plans
Improve password policies & password management system
Ensure full compliance with PCI DSS and GDPR
🔹 9. Conclusion

The audit revealed that Botium Toys has significant gaps in both technical and administrative security controls. While some basic protections exist, the organization must implement stronger security practices to protect sensitive data and maintain compliance with industry standards.

Risk assessment

Understanding of NIST CSF
Knowledge of compliance (PCI DSS, GDPR)
Security control analysis
